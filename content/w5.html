透過stud.cycu.org 執行動態網站 

  1. (登錄stud.cycu.org)利用@nfu電子郵箱接收到的賬號與密碼，登錄stud.cycu.org：在命令列表窗口，以ssh  cad學號@stud.cycu.org，輸入以應用，密碼可吅 
  2. 請到excel 檔案 查詢 stud.cycu.org 中與帳號對應的近端 (給127.0.0.1 使用) 與遠端 (給stud.cycu.org) 堡號 
  3. （sftp建立與 對應。    
  4. 與在近端 (Windows) 使用 SSH 建立公鑰/私鑰流程一致，必須在 Linux (Ubuntu) 使用 ssh-keygen 建立密鑰。所生成的id_rsa為私鑰，而乙id_rsa.pub為鑰.ssh目錄下。 
  5. 使用 sftp 進入 stud.cycu.org 後，cd 到 .ssh，以獲取 id_rsa.pub 將公鑰刪除至 Windows 後，送至 Github -> 設置 -> SSH 和 GPG 設置區，通過新密匙登錄錄。 
  6. 接下來要下載  config 設置文件，以sftp放入stud.cycu.org主機的.ssh目錄中。 
  7. 上面的config 主要設置SSH協議對github.com 連線時的代理主機, 但是是取消下cmsimde 子模型使用的是https, 因此執行下面的列表 git clone 時將有相同時使用到S config設置文件外，也必須同時在.gitconfig文件中設置http.proxy，也就是以git config --global http.proxy  http://p42.cycu.org:3128設置提供https連線使用的代理主機，並以 git config --global user.name 及 git config --global user.email 設置提供提供交互的身份記錄。  
  8. 在stud.cycu.org中，以git clone --recurse-submodules   git@github.com:mdecad2022/site-個人github賬號.git   取下個人倉庫。 
  9. 假如先使用 git clone 並沒有取下 cmsimde 子模型內容, 只需要在設置 https 代理主機後, 進入庫根目錄執行 git submodule update --init --recursive 即可. 
  10. 接著下載 server.py，在Windows下編寫server.py，將個人分配到的9xxxx填寫後保存文件，以sftp放入上列取下的庫根目錄。 
  11. 在stud.cycu.org系統中，進入倉庫（site-github賬號）後，執行python3 server.py，就可以啟動動態網站。 
  12. 設置完成後，各學生將會有一個廣域網上的動態網站（取代重新復制上的動態網站），  https: //stud.cycu.org:8xxxx 看個人的動態網站。 
  13. 動態改版後，轉為，必須必須靜態通過通過通過通過  ssh  ssh將傳回傳回傳回傳回傳回傳回傳回傳回。。。。（（必須使用使用使用使用使用使用使用使用++ 
  14. 執行source acp "提交字符串" 之前必須要 git config --global user.name,  git config --global user.email,  git config --global http.proxy 
  15. 上面所說的採用指令模式對服務器sftp也可以利用  Filezilla portable，以圖形介紹完成。而ssh也可以通過putty.exe執行遠程登錄。 

 

 

.ssh目錄下的配置： 

Host 就是 .ssh/config 設置的對應會話名稱，此處若 Host 設置為 demo，則 SSH 對應的 .git/config 中的 URL 就必須使用 git@demo:mdecad2022/site-scrum-1.git, demo 就是Host, Port 那那麼是隨後的 %p 也就是 SSH 協議內定的 port 22, %h 那那麼是 Hostname, 也就是 github.com
